写在开头

原本我的笔记内容都使用Notion进行管理，但是随着Notion公司越做越大，使用人数越来越多，我不禁担心起来未来个人用户收费的可能性。虽然目前notion免费个人用户仍然能够享受无限页面/块及最高5兆的文件上传容量，但是只给使用者提供云存储云编辑的选项还是不能完全满足我的需求。因此在很长时间弃用OB以后，我又重新在电脑上安装了Obsidian这款笔记管理软件。该软件旨在帮助用户以一种灵活的、非线性的方式来组织和构架他们的想法和知识。
什么是Obsidian?
Obsidian是一款使用Markdown语法的闭源笔记软件，用户可以在笔记之间建立双向链接（就像是互联网的超链接一样，不过双向链接是用户自己创建的，可以根据任何需求定制化双链）。Obsidian对于个人用户免费，对于商业用户收费。

5款自用的插件分享

Better Word Count

使用这个插件以后可以直接关闭OB自带的字数统计软件。

项目的Github地址：https://github.com/lukeleppan/better-word-count

与内置的笔记计数插件功能相同，但是在选中文字的时候，这个插件会计数选中的文字而不是整个文档，同时也拥有统计当天输出文字数量的功能。

效果演示如下：
![[Better Word Count.png]]

Docxer

轻松导入Docx文件，支持预览以及将Docx文件转换成Markdown文件。

Github地址：https://github.com/Developer-Mike/obsidian-docxer?tab=readme-ov-file#usage
使用方法：

  1. 将一个.docx 文件添加到库中
  2. 在OB中打开这个文件
  3. 点击画面上方出现的Convert按钮将docx文件转换为markdown文件。

效果演示：
下面以一个我以前完成的一份实验报告为例，展示转换效果，左图是Markdown文件，右图是Docx文件，可以看出转换效果非常不错，并且Docx文件中的所有图片会自动转换到MD文件所在的目录的下一级Attachments目录下（这个目录可以在插件设置中自己定义），并且使用了 ![]() 的MD语法引出。
![[Docxer使用效果.png]]

Contribution Graph

傻瓜式的生成热力图插件，代码小白友好，该插件的前置插件是Dataview插件。能够在指定页面生成一个类似于github profile中贡献图的页面。

项目地址：https://github.com/vran-dev/obsidian-contribution-graph
开发者：

使用方法：

  • 按下 Ctrl+p （Mac端Command+p)，输入 contribution 关键字检索命令 create contribution graph
  • 按下回车
  • 点击 save 按钮，图表就创建成功了
    效果演示：
    ![[Contribution Graph.png]]
    在生成以后可以直接点击右上角三条横线对显示样式进行更改。

Advanced Tables

github项目地址：https://github.com/tgrosinger/advanced-tables-obsidian
插件功能介绍：
在Obsidian中为 Markdown 表格添加了改进的导航、格式设置和操作：

  • 自动格式化
  • 类似于Excel的表格导航（在单元格之间按Tab建/回车键）
  • 方便地添加删除和移动行、列
  • 支持设置单元格对齐（左、中、右）
  • 按照指定的列对行进行排序
  • 导出为CSV
  • 适用于Obsidian移动端

演示效果：

使用教程：

快捷键 行为
Tab 下一个单元格
Shift + Tab 前一个单元格
Enter 下一行
Ctrl + Shift + D 打开Advanced Table 控制栏

Excalidraw

Obsidian上非常知名的绘图软件，相比于自带的白板软件，该软件的优势有：

  1. 手绘风格，简单排版就能获得出色的视觉效果。
  2. 支持手写笔绘制。
  3. 适合精细化绘图需求。
  4. 支持Openai 接口接入。

因为作者只使用了非常简单的该插件的功能，远不及全部功能的十分之一，在此附上github上面的中文项目说明：https://github.com/zsviczian/obsidian-excalidraw-plugin/blob/master/docs/zh-cn/README.md
